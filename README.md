# Priora v2.4.0 — المشروع الموحّد النهائي

هذا هو التسليم النهائي لمشروع إعادة الهيكلة الكاملة — كل الأقسام التسعة (1 إلى 9)
مدمجة بمشروع Vite واحد حقيقي، جاهز للتطوير والنشر.

## آخر تحديث (هذي الجلسة)

### 1. تصحيحات التواريخ
- تواريخ "التحديث/الملاحظة": أُزيل القيد العام على الماضي، بس بقي القيد المنطقي
  (ما قبل تاريخ إنشاء التاسك) — المستقبل ممنوع دائماً
- أُضيف قيد ناقص: تاريخ التاسك نفسه بمودال الإنشاء الآن max=اليوم
- أُضيف قيد ناقص آخر: تاريخ المتابعة بمودال EditTaskModal الآن min=اليوم

### 2. إغلاق المودالز بالضغط خارجها
- مودالز العرض (تفاصيل، سجل نشاط، حول..) تنقفل بالضغط برّاها
- مودالز الإدخال (إنشاء/تعديل) تبقى محمية، ما تنقفل بالغلط
- مودالز فيها تعديل inline تتعطّل الميزة مؤقتاً وقت التعديل النشط فقط

### 3. مجلدات فرعية + حذف آمن + لون متكيّف (ميزات جديدة)
- مستوى واحد من المجلدات الفرعية (رئيسي ← فرعي)
- حذف مجلد = نقل تاسكاته لسلة المهملات + المجلد نفسه يصير محذوف ناعم قابل للاستعادة
- استعادة أي تاسك ترجع مجلده معه تلقائياً لو كان محذوف
- زر Move to Folder بقائمة سياق التاسك
- لون المجلد يُحسب تلقائياً حسب الوضع الداكن/النهاري من لون أساسي واحد
- ترقية تلقائية لمرة واحدة: المجلدات الحالية تتحول لفروع تحت مجلد رئيسي جديد General

### 4. تصحيح خلل حقيقي
- expireTrash كانت تستخدم 24 ساعة ثابتة بدل الثابت المشترك (شهرين) — مُصحَّح الآن

## الفرق عن الزبّات السابقة (section-01 إلى section-09)

كل زب سابق كان **مشروع اختبار منفصل** (بـ TestHarness.jsx خاص بيه، ومخازن معطّلة عن
Firestore عمداً للاختبار الآمن). هذا المشروع **حقيقي بالكامل**:
- لا يوجد TestHarness — `App.jsx` هو نظام التنقل الفعلي (نفس منطق Jt/Vt بالأصل)
- المخازن تتصل فعلياً بـ Firestore (تحتاج مفاتيح حقيقية بملف `.env`)
- كل الملفات المكرّرة بين الأقسام (مثل authStore.js، Modal.jsx) دُمجت بنسخة واحدة
  لكل ملف (تأكدت من عدم وجود أي تعارض حقيقي بينها قبل الدمج)

## كيف تشغّله

```bash
npm install
cp .env.example .env
# عبّي .env بمفاتيح مشروع Firebase الحقيقي تبعك (نفس مشروع النسخة 2.3.0 المنشورة)
npm run dev
```

بدون تعبية `.env`، التطبيق **يتعلّق عند شاشة التحميل** (مش يطلع خطأ واضح) — لأن
الاتصال بـ Firebase بمفاتيح فاضية لا يفشل بسرعة، يقعد يحاول. هذا سلوك Firebase SDK
نفسه، وليس خلل بكودنا (لاحظنا نفس هذا السلوك بقسم Workspaces بمرحلة الاختبار).

## ما تم تأكيده

- `npm run build` نجح من أول محاولة — **151 ملف، صفر أخطاء** (هذا يثبت كل الاستيرادات
  بين الأقسام التسعة المدمجة سليمة 100%)
- `npm run dev` شغّال فعلياً، والأصول الثابتة (الشعار، manifest.json) تُخدَّم بشكل صحيح
- فحصت منطق التنقل (الفرق بين صفحة لها محتواها الخاص وصفحة تتحول لبحث شامل) بمحاكاة
  منفصلة، وتأكدت إنه يطابق سلوك الأصل بالضبط

## ⚠️ ما لم يُختبر فعلياً (يحتاج مفاتيح Firebase حقيقية)

- تسجيل الدخول الفعلي (بريد/Google)
- قراءة/كتابة البيانات الفعلية من Firestore
- الإشعارات الحقيقية بين أعضاء ورك سبيس
- أي تفاعل حقيقي بعد تسجيل الدخول

هذا متوقع ومقصود — كل هذا تم فحصه منطقياً بمعزل بكل قسم على حدة (بفحوصات تلقائية +
محاكاة Node)، بس التجربة الكاملة المتصلة فعلياً تحتاج مشروعك الحقيقي على Firebase.

## ملاحظات تصميمية ثلاث اتخذتها أثناء الدمج

1. **حذفت غلاف React Query (QueryClientProvider)** — كان موجود بالأصل لكن بدون أي
   استخدام فعلي لـ useQuery بأي مكان بالكود. حذفه تنظيف حقيقي، صفر تأثير على السلوك.
2. **حذفت سكربت إعادة التوجيه القديم** الموجود بـ index.html الأصلي (كان يحوّل أي
   دومين غير priora-io.vercel.app لذاك الموقع) — هذي بنية نشر خاصة بالدومين القديم،
   غير منطقية لنسخة جديدة تُنشر بمكان آخر.
3. **استخدمت الشعارات والأيقونات الحقيقية** المرفوعة بالزب الأصلي (logo-day.png,
   logo-night.png, manifest.json..) — موجودة بمجلد public/ هنا.
