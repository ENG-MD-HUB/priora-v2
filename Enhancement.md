كل سطر يحتوي على:

* دائرة الحالة
* اسم المشروع أو الطلب

* الحالة الحالية
* زر Update
* زر Complete

# نظام الـ Workspace

## الفكرة

المستخدم يستطيع إنشاء Workspace مشترك.

مهام الـ Workspace منفصلة عن المهام الشخصية.

الدخول يتطلب موافقة الـ Owner.

---

# إنشاء Workspace

الـ Owner يستطيع:

* اختيار الاسم
* إضافة وصف اختياري

ويتم إنشاء:
Invite Code تلقائي بصيغة:
XXXX-XXXX

---

# صلاحيات الـ Workspace

## Owner

* تعديل الاسم
* حذف الـ Workspace
* تجديد Invite Code
* قبول أو رفض الطلبات
* إزالة الأعضاء

## Member

* مشاهدة جميع المهام
* إضافة مهام
* تعديل مهامه فقط

---

# نظام الانضمام

1. المستخدم يضغط Join Workspace
2. يدخل Invite Code
3. يتم إرسال طلب
4. الـ Owner يرى إشعار مباشر
5. عند الموافقة يدخل المستخدم فوراً بدون Reload

# أنواع مهام الـ Workspace

## النوع الأول

New Workspace Task

* تُنشأ داخل الـ Workspace فقط
* يظهر اسم وصورة المنشئ
* لا تظهر في المهام الشخصية
* فقط المنشئ يستطيع تعديلها

---

## النوع الثاني

Shared Personal Task

* المستخدم يشارك مهمة شخصية
* تظهر في المكانين
* أي تعديل يتزامن فورياً
* فقط المالك الأصلي يستطيع التعديل أو الحذف

# Architecture مهمة جداً

استخدم Clean Architecture و Feature-based Structure.

لا تضع كل شيء داخل ملفات كبيرة.

يجب تقسيم المشروع بهذا الشكل:

src/

* app/
* shared/
* features/
* entities/
* services/
* hooks/
* store/
* components/
* pages/
* layouts/
* utils/
* types/

كل Feature يجب أن تكون مستقلة قدر الإمكان لتقليل الانكسارات أثناء التعديل.
